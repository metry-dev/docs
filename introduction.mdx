The Metry JavaScript SDK is the official browser client for sending events to your Metry project. You install it once, identify your users, and call `Metry.step()` at each point in your funnel. Metry handles batching, retries for transient failures, and page-lifecycle delivery paths so valid events can reach your dashboard with minimal ceremony — where you can see conversion, drop-off, step timing, and segment breakdowns without building anything yourself.

If you are new to Metry, the SDK is one half of the picture. The other half is the dashboard at [dashboard](https://dashboard.metry.dev), where your events turn into funnels, trends, and insights.

## What the SDK does

- **Identifies users.** Call `Metry.identify()` once you know who the user is, and every event after that is attached to them — along with any traits you pass (plan, source, email, and so on).
- **Tracks funnel steps.** Call `Metry.step()` at each step you want to measure. Metry builds the funnel on the server side from the step names you define in your dashboard.
- **Batches and flushes automatically.** Events queue in memory and flush every 5 seconds, or whenever the batch hits 50 events — whichever comes first. Both values are configurable.
- **Attempts delivery on page unload.** When a user closes the tab or navigates away, the SDK uses `navigator.sendBeacon` to hand the last batch to the browser.
- **Runs on the client.** The SDK has no import-time side effects, but call `init()` and tracking methods from browser/client code. Server runtimes skip browser lifecycle hooks, and modern server `fetch` implementations can still attempt network sends, so server-side tracking is not the supported integration path.

## What Metry does

The SDK is the ingest side. The dashboard is where the value lives:

- **Funnel overview** — step-by-step conversion, drop-off, and completion time for any funnel you define.
- **Trends** — daily movement and conversion rate over time, with filters for country, device, browser, OS, plan, and source.
- **Segments** — conversion performance broken down by user attribute, so you can see where a specific cohort is struggling.
- **User timelines** — the full chronological event history for any individual user, useful when you need to understand what actually happened.

You instrument with the SDK. Metry does the rest.

## Installation

Install the package from npm using your preferred package manager.

<CodeGroup>

```bash npm
npm install @metry-dev/sdk
```

```bash yarn
yarn add @metry-dev/sdk
```

```bash pnpm
pnpm add @metry-dev/sdk
```

</CodeGroup>
