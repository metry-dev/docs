---
title: "Quickstart"
description: "Install the Metry SDK, call init() with your API key, and send your first event. Setup takes about five minutes."
---

This guide walks you through installing the Metry SDK, wiring it into your app, and sending your first event. Events are queued in memory and sent in batches automatically — the default batch size is 50 (max 100), and the SDK flushes on a timer, when the batch fills, and on page unload.

Before you start, grab your publishable API key from **Settings → API Keys** in your Metry dashboard. You'll need it in step 2.

<Steps>

<Step title="Install the package">
Install `@metry-dev/sdk` from npm using your preferred package manager.

<CodeGroup>
```bash npm
npm install @metry-dev/sdk
```
```bash yarn
yarn add @metry-dev/sdk
```
```bash pnpm
pnpm add @metry-dev/sdk
```
</CodeGroup>
</Step>

<Step title="Initialize the SDK">
Import `Metry` and call `init()` once when your application boots. `apiKey` is required.

```ts
import { Metry } from '@metry-dev/sdk'

Metry.init({
  apiKey: 'pk_your_publishable_key'
})
```

<Warning>
  `Metry.init()` is browser-only. The package has no import-time side effects, but call `init()` and tracking methods from client-side code. In Next.js, call it from a Client Component — see the [Next.js guide](/guides/nextjs) for the recommended `<MetryProvider>` pattern.
</Warning>

<Tip>
  Pass `debug: true` during development to log SDK lifecycle messages, such as the resolved configuration on init and session resets. Warnings are logged regardless of this setting.

```ts
  Metry.init({
    apiKey: 'pk_your_publishable_key',
    debug: true,
  })
```
</Tip>
</Step>

<Step title="Identify the user">
Once the user is known — usually after login — call `Metry.identify()` to attach all subsequent events to them. Any traits you pass are included with every event that follows.

```ts
Metry.identify('user_123', {
  email: 'alice@example.com',
  plan: 'pro',
})
```

The identity is held in memory for the current page. Call `Metry.identify()` again to update traits, or `Metry.reset()` on logout to clear it.
</Step>

<Step title="Track your first event">
Call `Metry.step()` to record a named event. The first argument is the step name (1–100 characters, matching a step in one of your funnels). The second is an optional object of properties you want to attach.

```ts
Metry.step('signup_completed', {
  plan: 'pro',
  source: 'google_ads',
})
```

The event is queued immediately and sent in the next flush.
</Step>

<Step title="Let flushing happen automatically">
You don't need to manage flushing yourself. The SDK sends queued events:

- Every 5 seconds, on a timer
- As soon as the batch hits its size threshold (default 50 events)
- On page hide, using `navigator.sendBeacon`, so the browser can send the final batch as the tab closes

If you ever need a send attempt at a specific moment — for example, right before a hard navigation in a custom flow — you can flush manually:

```ts
await Metry.flush()
```
</Step>

</Steps>
