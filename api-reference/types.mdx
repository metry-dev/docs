The Metry SDK is written in TypeScript and exports all of its public types. This page documents each type, its fields, and how the types relate to one another. You can import any of these types directly from the package for use in your own TypeScript code.

## Config

`Config` describes the full resolved configuration object used internally by the SDK after defaults have been applied. When calling `Metry.init()`, you pass an `InitConfig` — a subset of `Config` where `flushInterval`, `maxBatchSize`, and `debug` are optional.

```typescript
export type Config = {
  apiKey: string
  endpoint: string
  flushInterval: number
  maxBatchSize: number
  debug?: boolean
}
```

<ResponseField name="apiKey" type="string" required>
  Your publishable API key. Included in every outbound request as the `x-api-key` header and also embedded in the JSON body as `EventRequestPayload.apiKey`.
</ResponseField>

<ResponseField name="endpoint" type="string" required>
  The HTTPS URL of your ingest endpoint. `http://` is only permitted for `localhost` addresses.
</ResponseField>

<ResponseField name="flushInterval" type="number" required>
  Milliseconds between automatic flushes. Defaults to `5000` when set via `Metry.init()`.
</ResponseField>

<ResponseField name="maxBatchSize" type="number" required>
  Maximum events per outbound batch. Defaults to `50`; hard-capped at `100` by the SDK.
</ResponseField>

<ResponseField name="debug" type="boolean">
  When `true`, verbose `[Metry]` messages are written to the browser console. Defaults to `false`.
</ResponseField>

## Identity

`Identity` holds the current user context that is attached to every event payload. It is updated by `Metry.identify()` and cleared by `Metry.reset()`.

```typescript
export type Identity = {
  userId: string | null
  traits?: Record<string, unknown>
}
```

<ResponseField name="userId" type="string | null" required>
  The identifier of the currently known user. `null` when no user has been identified (anonymous session).
</ResponseField>

<ResponseField name="traits" type="Record<string, unknown>">
  Arbitrary key-value metadata about the user. Set by calling `Metry.identify()`. Persists across events until overwritten or cleared by `Metry.reset()`.
</ResponseField>

## StepEvent

`StepEvent` is the shape of a single tracked event as it exists in the internal queue and inside the `events` array of an outbound payload.

```typescript
export type StepEvent = {
  name: string
  properties?: Record<string, unknown>
  timestamp: string
}
```

<ResponseField name="name" type="string" required>
  The event name provided to `Metry.step()`. Between 1 and 100 characters.
</ResponseField>

<ResponseField name="properties" type="Record<string, unknown>">
  Optional plain-object metadata provided to `Metry.step()`. Present only when the caller supplied properties.
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 timestamp (`new Date().toISOString()`) recorded at the moment `Metry.step()` was called, before the event enters the queue.
</ResponseField>

## EventRequestPayload

`EventRequestPayload` is the complete JSON object sent to your ingest endpoint in the body of each POST request. The `Payload` type exported from the SDK is an alias for `EventRequestPayload`.

```typescript
export type EventRequestPayload = {
  type: 'event'
  apiKey: string
  sessionId: string
  identity: Identity
  events: StepEvent[]
  sentAt: string
}

export type Payload = EventRequestPayload
```

<ResponseField name="type" type='"event"' required>
  A discriminant literal. Always `"event"`. Reserved for future payload type variants.
</ResponseField>

<ResponseField name="apiKey" type="string" required>
  The `apiKey` from your `Config`. Included in the body in addition to the `x-api-key` request header for beacon requests, which cannot carry custom headers.
</ResponseField>

<ResponseField name="sessionId" type="string" required>
  A randomly generated identifier for the current browser session. Regenerated by `Metry.reset()` and when the SDK detects a corrupt value.
</ResponseField>

<ResponseField name="identity" type="Identity" required>
  A snapshot of the current `Identity` (userId + traits) at the time the batch is flushed.
</ResponseField>

<ResponseField name="events" type="StepEvent[]" required>
  The batch of events being delivered. Contains between 1 and `maxBatchSize` (capped at 100) `StepEvent` objects.
</ResponseField>

<ResponseField name="sentAt" type="string" required>
  ISO 8601 timestamp recorded at the moment the payload is serialised and sent, distinct from the per-event `timestamp` fields inside `events`.
</ResponseField>

---

<Note>
  All timestamp fields (`StepEvent.timestamp` and `EventRequestPayload.sentAt`) use `new Date().toISOString()`, which produces UTC strings in the format `2024-01-15T12:34:56.789Z`.
</Note>
